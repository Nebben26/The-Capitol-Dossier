# Morning Brief — Setup Guide

A daily email digest sent at 7am ET to subscribers. Built on Resend + Supabase + Netlify scheduled functions.

---

## 1. Resend account

1. Sign up at resend.com
2. Add and verify your sending domain (e.g., `quivermarkets.com`)
3. Create an API key with "Sending access"
4. Copy the key — you'll need it in step 3

---

## 2. Supabase — run migrations

Run the following SQL in the Supabase SQL editor (Settings → SQL Editor):

```sql
-- File: scripts/migrations/session9-morning-brief.sql
```

This creates:
- `morning_brief_subscribers` — email list with unsubscribe tokens
- `morning_brief_sends` — send log (one row per blast)

---

## 3. Environment variables

Add these to Netlify (Site → Settings → Environment Variables):

| Variable | Value | Notes |
|---|---|---|
| `RESEND_API_KEY` | `re_...` | From Resend dashboard |
| `MORNING_BRIEF_FROM_EMAIL` | `brief@quivermarkets.com` | Must match verified domain |
| `MORNING_BRIEF_CRON_SECRET` | any random string | Protects the send endpoint |
| `NEXT_PUBLIC_SITE_URL` | `https://quivermarkets.com` | Used in unsubscribe links |

Generate a secure secret: `openssl rand -hex 32`

---

## 4. Install Netlify Functions package (once)

```bash
npm install --save-dev @netlify/functions
```

This enables TypeScript types for the scheduled function in `netlify/functions/morning-brief-cron.ts`.

---

## 5. How it works

```
7:00am ET daily
  └─ Netlify cron fires morning-brief-cron.ts
       └─ POST /api/morning-brief/send  (Bearer MORNING_BRIEF_CRON_SECRET)
            ├─ generateMorningBrief()   queries disagreements + movers from Supabase
            ├─ renderMorningBriefHtml() renders dark-themed HTML email
            └─ sends via Resend to all active subscribers (batches of 10)
                 └─ logs results to morning_brief_sends
```

## 7. Subscription flow

| Action | Endpoint | Result |
|---|---|---|
| Subscribe | `POST /api/morning-brief/subscribe` | `{ "email": "..." }` |
| Unsubscribe | `GET /api/morning-brief/unsubscribe?token=<token>` | Redirect to `/morning-brief?unsubscribed=true` |
| Preview email | `GET /api/morning-brief/preview` | Returns HTML |

Unsubscribe tokens are UUIDs auto-generated by Supabase (`gen_random_uuid()`). They're included in every email footer.

---

## 8. Cron schedule

The function in `netlify/functions/morning-brief-cron.ts` runs on:
```
0 11 * * *   →  11:00 UTC = 7:00am Eastern
```

To change the time, edit the `schedule` value in that file and redeploy.

## 10. Troubleshooting

**"Resend not configured"** — `RESEND_API_KEY` env var is missing or still contains "PLACEHOLDER"

**"Unauthorized"** — `Authorization: Bearer` header doesn't match `MORNING_BRIEF_CRON_SECRET`

**Email goes to spam** — verify your sending domain in Resend; ensure SPF/DKIM records are set

**Unsubscribe link invalid** — the token in the URL doesn't match any row; check `morning_brief_subscribers` table
